# SVETOZAR SCHOOL

Образовательная платформа (LMS): курсы → модули → уроки, собственный видеоплеер,
ДЗ с проверкой ИИ-наставником, геймификация (XP, стрики, достижения, рейтинг),
профиль, центр знаний, интеграция с Telegram и админ-панель.

Правила разработки — в [CLAUDE.md](./CLAUDE.md), архитектура — в [ARCHITECTURE.md](./ARCHITECTURE.md),
требования — в [TZ_LMS_Platform.md](./TZ_LMS_Platform.md).

## Стек
Next.js 15 (App Router) · TypeScript (strict) · PostgreSQL + Prisma · Auth.js (NextAuth v5) ·
Tailwind + дизайн-токены (чёрный + золото) · Anthropic SDK · Bunny Stream · Telegram (grammY) · Zod.

## Быстрый старт
```bash
npm install
cp env.example .env.local      # заполнить DATABASE_URL, AUTH_SECRET и т.д.
npm run db:generate            # сгенерировать Prisma Client
npm run db:migrate             # применить миграции (нужна БД PostgreSQL)
npm run db:seed                # засеять админа, курсы, достижения, статьи
npm run dev                    # http://localhost:3000
```

Демо-доступы после сидинга: `admin@svetozar.school` / `ChangeMe_2026!`,
`student@svetozar.school` / `Student_2026!` (удалить перед продом — ТЗ §6А.14).

### 2FA (двухфакторная аутентификация)
- Включается в профиле (раздел «Безопасность»): QR-код для приложения-аутентификатора + подтверждение кодом.
- Секрет TOTP хранится в БД в зашифрованном виде (AES-256-GCM, ключ из `AUTH_SECRET`).
- При входе с включённой 2FA требуется 6-значный код (поле в форме входа).
- **Обязательна для доступа в админ-панель**: админа без 2FA редиректит в профиль для подключения.

## Деплой (Vercel + Neon)
1. Создать БД PostgreSQL (Neon free) → скопировать `DATABASE_URL`.
2. В Vercel завести проект из репозитория; в Environment Variables задать все ключи из `env.example` (минимум: `DATABASE_URL`, `AUTH_SECRET`, `NEXT_PUBLIC_APP_URL`). Серверные секреты — без префикса `NEXT_PUBLIC_`.
3. Применить миграции и сид: `npx prisma migrate deploy && npm run db:seed` (или через Vercel build hook).
4. Перед открытием доступа реальным ученикам (security gate, ТЗ §6А.14): удалить seed-аккаунты, включить 2FA админа, проверить HTTPS/HSTS и заголовки, прогнать `npm audit`.
