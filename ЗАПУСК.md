# 🚀 Запуск SVETOZAR SCHOOL — пошаговый план вывода MVP в прод

> Код готов (этапы 0–7 + 2FA). Этот документ — что сделать руками, чтобы платформа заработала

Заведи аккаунты (если ещё нет). Везде включи 2FA на сам аккаунт.

1. **GitHub** — хранение кода и автодеплой. github.com
2. **Vercel** — хостинг приложения. vercel.com (войти через GitHub). 💲 Hobby = $0

6. **Resend** — письма (подтверждение email, сброс пароля). resend.com. 💲 free tier = $0
7. **Telegram** — бота создаём в самом Telegram через **@BotFather** (см. этап F).
8. *(опц.)* **Домен** — например, svetozar.school. 💲 ~$12/год
9. *(опц., но желательно)* **Sentry** (ошибки) + **UptimeRobot** (аптайм) — free.

## Этап C. База данных Neon (15 мин)

1. В Neon создай проект (регион ближе к основной аудитории) → скопируй **Connection string** (Pooled). 🔑 `DATABASE_URL`
2. Локально открой `.env` и вставь реальный `DATABASE_URL`.
3. Сгенерируй `AUTH_SECRET`: 🔑
   ```bash
   node -e "console.log(require('crypto').randomBytes(32).toString('base64'))"
   ```
   Вставь в `.env` как `AUTH_SECRET`.
4. Создай первую миграцию и примени её (папки `prisma/migrations` ещё нет — она создастся сейчас):
   ```bash
   npm run db:migrate -- --name init     # prisma migrate dev: создаёт и применяет миграцию
   ```
5. Засей стартовые данные (админ, 2 курса, достижения, статьи):
   ```bash
   npm run db:seed
   ```
   Запиши логин админа из вывода (по умолчанию `admin@svetozar.school` / `ChangeMe_2026!`).
   > Пароль админа можно задать заранее: `SEED_ADMIN_EMAIL` и `SEED_ADMIN_PASSWORD` в `.env` перед сидом.
6. Проверь локально: `npm run dev` → http://localhost:3000 → войди админом.

> ⚠️ Закоммить созданную папку `prisma/migrations` в git (она нужна для прод-деплоя):
> `git add prisma/migrations && git commit -m "init migration" && git push`

## Этап D. Ключи внешних сервисов (40–60 мин)

Заполняй `.env` по мере получения. Серверные секреты — БЕЗ префикса `NEXT_PUBLIC_`.

### D1. Anthropic (Claude) 🔑
- console.anthropic.com → API Keys → Create Key → `ANTHROPIC_API_KEY`.
- Модель оставь `ANTHROPIC_MODEL=claude-haiku-4-5-20251001` (дёшево/быстро для проверки ДЗ).
- Пополни баланс ($5 хватит надолго на нашем масштабе).

### D2. Bunny Stream (видео) 🔑
- bunny.net → **Stream** → Create Video Library → запиши **Library ID** → `BUNNY_LIBRARY_ID`.
- В библиотеке: **API** → `BUNNY_API_KEY` (ключ библиотеки, для загрузки/удаления).
- **Security** → включи **Token Authentication** → скопируй ключ → `BUNNY_TOKEN_AUTH_KEY`.

### D3. Bunny Storage (файлы-вложения .pdf/.docx) 🔑
- bunny.net → **Storage** → Add Storage Zone → имя → `BUNNY_STORAGE_ZONE`.
- Пароль зоны (FTP & API password) → `BUNNY_STORAGE_PASSWORD`.

### D4. Resend (email) 🔑
- resend.com → API Keys → `RESEND_API_KEY`.
- Domains → добавь домен и подтверди DNS-записи (или для теста используй их sandbox-домен).
- `EMAIL_FROM=noreply@<твой-домен>`.

### D5. Telegram — см. этап F (после деплоя, нужен публичный URL).

## Этап E. Деплой на Vercel (20 мин)

1. vercel.com → **Add New → Project** → импортируй репозиторий с GitHub.
2. Framework = Next.js (определится сам). Build Command по умолчанию (`next build`) —
   `prisma generate` уже выполнится через `postinstall`.
3. **Environment Variables** — добавь ВСЕ из `.env` (кроме комментариев):
   `DATABASE_URL`, `AUTH_SECRET`, `NEXTAUTH_URL`, `NEXT_PUBLIC_APP_URL`,
   `ANTHROPIC_API_KEY`, `ANTHROPIC_MODEL`,
   `BUNNY_API_KEY`, `BUNNY_LIBRARY_ID`, `BUNNY_TOKEN_AUTH_KEY`, `BUNNY_STORAGE_ZONE`, `BUNNY_STORAGE_PASSWORD`,
   `RESEND_API_KEY`, `EMAIL_FROM`,
   `TELEGRAM_BOT_TOKEN`, `TELEGRAM_WEBHOOK_SECRET`, `TELEGRAM_BOT_USERNAME` (после этапа F).
   - `NEXTAUTH_URL` и `NEXT_PUBLIC_APP_URL` = твой прод-URL (напр. `https://svetozar.school`).
4. **Deploy**. Дождись зелёной сборки.
5. **Применить миграции к прод-БД** (один раз и при каждом изменении схемы):
   ```bash
   # локально, с прод DATABASE_URL в окружении:
   DATABASE_URL="<neon-prod-url>" npm run db:migrate:deploy
   DATABASE_URL="<neon-prod-url>" npm run db:seed   # первый раз — засеять
   ```
   *(Альтернатива: добавить `prisma migrate deploy` в Build Command Vercel, но осознанно.)*
6. Открой прод-URL → проверь вход админом.
7. *(опц.)* Подключи домен: Vercel → Project → Domains. HTTPS/HSTS включены автоматически.

## Этап F. Telegram-бот (15 мин)

1. В Telegram напиши **@BotFather** → `/newbot` → задай имя и username.
   - Токен → `TELEGRAM_BOT_TOKEN` 🔑
   - Username (без @) → `TELEGRAM_BOT_USERNAME` 🔑
2. Придумай секрет вебхука 🔑 → `TELEGRAM_WEBHOOK_SECRET`:
   ```bash
   node -e "console.log(require('crypto').randomBytes(16).toString('hex'))"
   ```
3. Добавь эти 3 переменные в Vercel и **передеплой** (Redeploy).
4. Зарегистрируй вебхук (подставь свои значения):
   ```bash
   curl "https://api.telegram.org/bot<TELEGRAM_BOT_TOKEN>/setWebhook" \
     -d "url=https://<твой-домен>/api/telegram/webhook" \
     -d "secret_token=<TELEGRAM_WEBHOOK_SECRET>"
   ```
   Ответ `{"ok":true}` = готово.
5. Проверка: в профиле ученика «Подключить Telegram» → перейти к боту → должно прийти подтверждение.

## Этап G. Контент: видео и курсы (зависит от объёма)

1. **Видео**: загрузи ролики в Bunny Stream (UI или API). У каждого видео есть **GUID**.
2. В админке (`/admin/courses`) создай/отредактируй курсы → модули → уроки.
   В редакторе урока в поле «Видео» вставь **GUID** видео из Bunny.
3. Прикрепи материалы (.pdf/.docx) — *(загрузка вложений в UI — пункт фазы 2; сейчас файлы
   заводятся в Bunny Storage, ссылка кладётся в БД; для MVP можно без вложений)*.
4. **Обучение ИИ-агента** (`/admin/ai`): для каждого курса добавь тезисы/эталоны/критерии в базу
   знаний и выставь проходной балл и строгость — по ним наставник проверяет ДЗ.
5. **Достижения** (`/admin/achievements`) — стартовые засеяны; добавь свои при желании.

## Этап H. Первый админ и доступы (10 мин)

1. Войди админом → **смени пароль** в профиле (seed-пароль временный).
2. В профиле **включи 2FA** (обяз. для админки): отсканируй QR в Google Authenticator → подтверди код.
   После этого `/admin` доступен.
3. Выдавай ученикам доступ к курсам: `/admin/students` → карточка ученика → «Выдать доступ».
